# Athena MCP Tools - Complete Reference

**Package:** `@xylex-group/athena-mcp`
**Total Tools:** 169

This page provides **documentation for each and every tool**.

## General Behavior
- Optional `client` field when multiple `ATHENA_AVAILABLE_CLIENTS` configured.
- Read-only mode blocks all mutating operations.
- Full schemas in the package's `mcp-tools.json` and `athena-mcp-openapi.yaml`.

## Schema & Data Discovery

### `list_tables`

List all tables available in the connected PostgreSQL database. Primary schema discovery tool.

**Inputs:**
See full schema.

**Read-only safe:** Yes

**Backend:** /schema/*

---

### `list_schemas`

List database schemas visible to the current Athena client

**Inputs:**
- `include_system` (boolean, optional): Include pg_catalog and information_schema schemas

**Read-only safe:** Yes

**Backend:** /schema/*

---

### `list_extensions`

List all installed PostgreSQL extensions. May return empty if the gateway does not expose extension metadata.

**Inputs:**
See full schema.

**Read-only safe:** Yes

**Backend:** Athena HTTP API or @xylex-group/athena SDK

---

### `list_views`

List visible views (and optionally materialized views). Uses Athena schema API.

**Inputs:**
- `schema` (string, optional): Schema to limit the view lookup to
- `include_materialized` (boolean, optional): Include materialized views (when supported by schema API)

**Read-only safe:** Yes

**Backend:** /schema/*

---

### `get_columns_of_table`

Describe columns for a table using Athena's schema API

**Inputs:**
- `table` (string, required): Table name (optionally schema-qualified) to describe
- `schema` (string, optional): Optional schema name when the table name is not schema-qualified

**Read-only safe:** Yes

**Backend:** /schema/*

---

### `list_table_metadata`

Return the full metadata for a table: schema name, table name, and each column's name, type, default value, and nullable flag

**Inputs:**
- `table` (string, required): Table name (optionally schema-qualified)
- `schema` (string, optional): Optional schema name when the table name is not schema-qualified

**Read-only safe:** Yes

**Backend:** Athena HTTP API or @xylex-group/athena SDK

---

### `list_all_table_metadata`

Return metadata for all tables in one call: schema, name, columns, types, defaults, nullable. Uses Athena schema API.

**Inputs:**
- `schema` (string, optional): Optional schema to limit to (default: all user schemas)

**Read-only safe:** Yes

**Backend:** Athena HTTP API or @xylex-group/athena SDK

**Read-only safe:** Yes

## Row Access & Basic CRUD

### `get_row_by_id`

Fetch rows by primary key column value. Simplifies the common fetch-by-id use case.

**Inputs:**
- `table` (string, required): Table name (optionally schema-qualified)
- `id` (string|number|boolean|null, required): Primary key value (typically id)
- `id_column` (string, optional): Primary key column name (defaults to 'id')
- `schema` (string, optional): Optional schema when table name is not schema-qualified
- `limit` (integer, optional): Maximum rows to return (defaults to 100)

**Read-only safe:** Yes

**Backend:** Athena HTTP API or @xylex-group/athena SDK

---

### `get_row_by_eq_column_of_table`

Fetch rows from a table where `column = value` using Athena's fetch endpoint

**Inputs:**
- `table` (string, required): Table name to query (optionally schema-qualified)
- `column` (string, required): Column name to match against
- `value` (string|number|boolean|null, required): Value to compare (converted to string for Athena)
- `schema` (string, optional): Optional schema to override the table name
- `limit` (integer, optional): Maximum number of rows to return (defaults to 100)

**Read-only safe:** Yes

**Backend:** Athena HTTP API or @xylex-group/athena SDK

---

### `insert_row`

Insert a row into a table. Blocked when read_only mode is enabled.

**Inputs:**
- `table` (string, required): Table name (optionally schema-qualified)
- `data` (object, required): Row data as key-value pairs
- `schema` (string, optional): Optional schema when table name is not schema-qualified

**Read-only safe:** No — blocked when READ_ONLY=true

**Backend:** /gateway/* or /query/*

---

### `update_row`

Update rows matching a condition. Blocked when read_only mode is enabled.

**Inputs:**
- `table` (string, required): Table name (optionally schema-qualified)
- `set` (object, required): Column-value pairs to set
- `where_column` (string, required): Column to match in WHERE clause
- `where_value` (string|number|boolean|null, required): Value to match (converted to string)
- `schema` (string, optional): Optional schema when table name is not schema-qualified

**Read-only safe:** No — blocked when READ_ONLY=true

**Backend:** /gateway/* or /query/*

---

### `delete_row`

Delete a row by primary key (resource_id). Blocked when read_only mode is enabled.

**Inputs:**
- `table` (string, required): Table name (optionally schema-qualified)
- `resource_id` (string, required): Primary key value of the row to delete
- `schema` (string, optional): Optional schema when table name is not schema-qualified

**Read-only safe:** No — blocked when READ_ONLY=true

**Backend:** /gateway/* or /query/*
